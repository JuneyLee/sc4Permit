# V1.6 주요 변경 사항
# 프로젝트 변경 사항 (Revisions)

## 📅 업데이트 날짜: 2026-03-25

### 1. Google Apps Script (GAS) 백엔드 구축 (Code.gs)
*   **보안 강화:** 클라이언트 사이드에서 직접 호출하던 Gemini API를 GAS 백엔드(`doPost`)로 이전하여 API 키 노출을 방지하고 보안을 강화했습니다.
*   **프록시 서버 역할:** `Code.gs`가 브라우저와 Gemini API 사이의 중계자(Proxy) 역할을 수행하여 CORS 이슈를 해결하고 에러 처리를 중앙 집중화했습니다.
*   **동적 페이지 서빙:** `doGet` 함수를 통해 `index`, `AndApp`, `4SC` 등 여러 HTML 파일을 하나의 웹 앱 URL에서 파라미터(`?page=...`)로 서빙할 수 있도록 구조를 개선했습니다.

### 2. Gemini API 에러 해결 및 자동화 (Code.gs)
*   **404 Not Found 해결:** 2026년 기준 지원되지 않는 구형 모델명 대신, 현재 사용 가능한 최신 모델을 자동으로 찾아주는 `getLatestFlashModel()` 함수를 도입했습니다.
*   **엔드포인트 최적화:** `v1beta` 엔드포인트를 사용하여 최신 기능 및 모델과의 호환성을 확보했습니다.
*   **중복 호출 제거:** 불필요하게 API를 두 번 호출하던 로직을 정리하여 할당량 낭비를 방지했습니다.
*   **디버깅 도구 추가:** `debugListModels()` 함수를 추가하여 API 키로 접근 가능한 모델 목록을 언제든 확인할 수 있게 했습니다.

### 2. 한도 초과(429) 안내 기능 추가 (AndApp.html)
*   **사용자 안내 문구:** 분석 결과 레이블 옆에 `(1분 뒤 다시 시도해 주세요)`라는 안내 문구를 추가했습니다.
*   **자동 감지:** Gemini API에서 `429 Quota Exceeded` 에러가 발생할 경우에만 해당 문구가 주황색으로 나타나도록 구현했습니다.
*   **초기화 로직:** 새로운 분석을 시작하면 안내 문구가 자동으로 사라집니다.

### 3. UI 및 사용성 개선 (AndApp.html)
*   **에러 확인 시간 연장:** 에러 발생 시 토스트 알림이 너무 빨리 사라지는 문제를 해결하기 위해, 에러 메시지 노출 시간을 **30초**로 연장했습니다. (일반 메시지는 5초 유지)
*   **상세 에러 로깅:** 에러 발생 시 구글 시트에 기록되는 내용을 상세화하여 문제 발생 시 원인 파악이 쉽도록 개선했습니다.

### 4. 아이폰용 페이지 유지 (4SC.html)
*   **기존 기능 보존:** 사용자의 요청에 따라 아이폰 단축어 연동에 최적화된 `4SC.html`의 토스트 노출 시간(2초) 및 기존 로직을 그대로 유지했습니다.

---
**⚠️ 주의사항:** 모든 서버 측 변경 사항(`Code.gs`)은 구글 스크립트 에디터에서 **[새 버전으로 배포]**를 완료해야 실제 앱에 반영됩니다.



# v1.5 주요 변경 사항

## PDF 용량 최적화
흑백(스캔) 모드 PDF 생성 시 이미지 압축률 조정 (JPEG Quality 0.8)
흑백 변환 후 용량이 비정상적으로 커지던 문제 해결 및 가벼운 파일 생성 가능

## 화면 출력 안정화
업데이트 후 발생했던 '화면 백화 현상(빈 화면)' 수정
리액트 컴포넌트 렌더링 로직 보강으로 안드로이드/아이폰 환경 모두 정상 작동

## 이미지 처리 보강
이미지 변환 중 발생할 수 있는 오류 예외 처리 추가
이미지 로딩 실패 시에도 앱이 멈추지 않도록 안정성 강화

## 보안 및 편의성 강화
Gemini API 키를 서버에서 안전하게 관리하여 키 노출 위험 제거
사용자 별도 입력 없이 즉시 분석 기능 사용 가능
